# AeroPoint MVP Design

## Goal

AeroPoint is a same-Wi-Fi iPhone app for controlling a Mac without a physical keyboard or mouse. The MVP supports mouse simulation and keyboard simulation through a native iPhone app paired with a native macOS menu bar agent.

## Scope

The first version is Mac-first and local-network only. It does not include cloud accounts, internet relay access, Windows support, gesture air-mouse control, screen streaming, file transfer, or multi-computer fleet management.

## Architecture

AeroPoint has two independently buildable applications:

- **iPhone app:** SwiftUI app that handles pairing, connection state, touchpad gestures, keyboard controls, and command serialization.
- **macOS agent:** Swift menu bar app that exposes a local WebSocket server, manages pairing, checks Accessibility permission, authenticates paired clients, and injects macOS mouse and keyboard events.

The iPhone app discovers or connects to the Mac on the same Wi-Fi network. The Mac agent displays a QR code containing the Mac host, port, pairing nonce, and server identity. The iPhone scans the QR code, completes pairing, stores a shared token in Keychain, and uses that token for future WebSocket sessions.

## User Experience

The Mac user launches AeroPoint Agent from the menu bar and grants Accessibility permission when prompted. The menu bar popover shows connection status, local address, pairing QR code, and currently connected device.

The iPhone user opens AeroPoint, scans the QR code, and lands on a controller view. The controller view has a full-screen touchpad area and a keyboard panel. The touchpad supports pointer movement, left click, right click, drag, and scroll. The keyboard panel supports text entry, common keys, and Mac modifier combinations.

## Frontend Responsibilities

The iPhone app owns:

- Pairing QR scan and manual host entry.
- Connection lifecycle and reconnect UI.
- Touch gesture recognition.
- Keyboard UI and key mapping.
- Outbound command serialization.
- Local storage of trusted Mac profiles and tokens.

## Backend Responsibilities

The macOS agent owns:

- Menu bar lifecycle and status UI.
- Pairing QR generation and token creation.
- Local WebSocket server.
- Client authentication and session validation.
- Accessibility permission detection and prompt routing.
- Translation from AeroPoint protocol messages to macOS input events.

## Protocol

Messages are JSON over WebSocket. Every command includes a monotonically increasing sequence number so the agent can ignore duplicates if reconnects occur.

Example commands:

```json
{ "seq": 101, "type": "mouse.move", "dx": 12, "dy": -4 }
{ "seq": 102, "type": "mouse.click", "button": "left" }
{ "seq": 103, "type": "mouse.scroll", "dx": 0, "dy": -80 }
{ "seq": 104, "type": "keyboard.text", "text": "hello" }
{ "seq": 105, "type": "keyboard.key", "key": "Enter", "modifiers": ["Command"] }
```

Connection messages:

```json
{ "type": "hello", "clientId": "iphone-uuid", "token": "paired-token" }
{ "type": "hello.ok", "serverName": "Wany's Mac", "protocolVersion": 1 }
{ "type": "error", "code": "accessibility_permission_missing" }
```

## Security

The MVP stays on the local network but still requires pairing. The Mac agent must reject unauthenticated WebSocket clients. Pairing tokens are generated by the Mac agent, shown only through the QR code or manual pairing flow, and stored in Keychain on both sides where available.

The Mac agent should bind only to the local network interface for MVP development. It should show visible connection state while a phone is connected, and it should provide a clear disconnect and unpair action.

## Error Handling

The iPhone app shows actionable states for unavailable Mac, failed pairing, disconnected socket, invalid token, and unsupported protocol version.

The macOS agent reports missing Accessibility permission, blocked local network access, invalid messages, authentication failure, and input injection errors. Invalid command payloads are rejected without crashing the agent.

## Testing Strategy

Frontend tests cover command encoding, keyboard mapping, touch gesture translation, and connection state transitions.

Backend tests cover protocol decoding, authentication, command validation, and input event routing through a mock input injector. Manual QA verifies real Accessibility permission and pointer/keyboard injection on macOS.

End-to-end testing can start with a local simulator client before the full iPhone UI is complete.

## Future Work

Planned later additions:

- Gesture air-mouse mode using iPhone motion sensors.
- Windows desktop agent.
- Internet relay mode.
- Multiple saved computers.
- Custom shortcuts and app-specific control layouts.
- Screen preview or low-latency screen streaming.
